# Руководство по запуску проекта

Пошаговая инструкция для полного запуска системы с нуля — от прошивки Arduino до деплоя VPS.

---

## Предварительные требования

| Компонент | Что установить |
|---|---|
| Raspberry Pi 4 | Docker, Docker Compose, Node.js 20+, статический IP в локальной сети |
| Компьютер для прошивки | Arduino IDE 2.x |
| VPS | Docker, Docker Compose, зарегистрированный домен |
| Роутер | Все устройства (Arduino, Raspberry) в одной WiFi-сети |

---

## Шаг 1 — Клонировать репозиторий

```bash
git clone <url-репозитория>
cd IOT_water_final
```

---

## Шаг 2 — Настроить статический IP Raspberry Pi

На Raspberry Pi добавить в `/etc/dhcpcd.conf`:

```
interface eth0
static ip_address=192.168.1.100/24
static routers=192.168.1.1
static domain_name_servers=8.8.8.8
```

Перезапустить сетевой сервис:

```bash
sudo systemctl restart dhcpcd
```

Проверить IP:

```bash
hostname -I
```

## Шаг 3 — Прошить Arduino Nano #1 (насосы + уровень)

### 3.1 Установить библиотеки в Arduino IDE

Открыть **Tools → Manage Libraries**, установить:
- `NewPing` by Tim Eckel
- `ArduinoJson` by Benoit Blanchon (версия 6.x)

### 3.2 Настроить параметры подключения

Открыть `arduino/nano1_pumps/nano1_pumps.ino`, отредактировать:

```cpp
#define WIFI_SSID      "НазваниеВашейСети"
#define WIFI_PASS      "ПарольВашейСети"
#define SERVER_HOST    "192.168.1.100"    // IP Raspberry Pi
#define SERVER_PORT    8000
#define STATION_ID     "station_001"
```

### 3.3 Загрузить прошивку

1. Подключить Arduino Nano к компьютеру по USB
2. В Arduino IDE: **Tools → Board → Arduino AVR Boards → Arduino Nano**
3. **Tools → Processor → ATmega328P (Old Bootloader)** (если плата не распознаётся)
4. Выбрать правильный порт: **Tools → Port**
5. Нажать **Upload** (Ctrl+U)
6. Открыть **Serial Monitor** (9600 бод) — должны появиться сообщения о подключении к WiFi

## Шаг 4 — Прошить ESP-32S (датчик влажности)

ESP-32S имеет **встроенный WiFi** — не нужен отдельный ESP8266, SoftwareSerial и делитель напряжения.

### 4.1 Добавить пакет плат ESP32 в Arduino IDE

В Arduino IDE открыть **File → Preferences**, в поле «Additional boards manager URLs» добавить:

```
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
```

Затем открыть **Tools → Board → Boards Manager**, найти `ESP32 by Espressif Systems`, установить.

### 4.2 Установить библиотеки в Arduino IDE

Открыть **Tools → Manage Libraries**, установить:
- `arduinoWebSockets` by Markus Sattler
- `ArduinoJson` by Benoit Blanchon (версия 6.x)

### 4.3 Настроить параметры подключения

Открыть `arduino/esp32_moisture/esp32_moisture.ino`, отредактировать:

```cpp
#define WIFI_SSID      "НазваниеВашейСети"
#define WIFI_PASS      "ПарольВашейСети"
#define SERVER_HOST    "192.168.1.100"    // IP Raspberry Pi
#define SERVER_PORT    8000
#define STATION_ID     "station_001"
```

### 4.4 Загрузить прошивку

1. Подключить ESP-32S к компьютеру по USB
2. В Arduino IDE: **Tools → Board → ESP32 Arduino → ESP32 Dev Module** (или соответствующую модель)
3. **Tools → Upload Speed → 115200**
4. Выбрать правильный порт: **Tools → Port**
5. Нажать **Upload** (Ctrl+U)
6. Открыть **Serial Monitor** (**115200 бод**) — должны появиться сообщения о подключении к WiFi и WebSocket

### 4.5 Откалибровать датчик влажности

ESP-32S использует 12-бит АЦП (диапазон 0–4095, вместо 0–1023 на Arduino):

1. Открыть **Serial Monitor** (115200 бод)
2. Дождаться строки `[STATS]` (появляется каждые 60 сек) — записать `Last raw moisture` при **сухом** датчике → это `DRY_VALUE` (обычно ~2800)
3. Поднести датчик к мокрой трубке / опустить в воду → записать `Last raw moisture` → это `WET_VALUE` (обычно ~1400)
4. Вписать значения в `.ino` и перепрошить:

```cpp
#define DRY_VALUE  2800   // ваше значение (12-бит АЦП)
#define WET_VALUE  1400   // ваше значение (12-бит АЦП)
```

> **Примечание:** В отличие от Arduino Nano, ESP-32S можно питать прямо от USB.
> Питать датчик влажности можно от пина 3.3V платы ESP-32S — встроенного LDO достаточно.

## Шаг 5 — Запустить Raspberry Pi (API + БД)

### 5.1 Скопировать и заполнить `.env`

```bash
cd raspberry
cp .env.example .env
nano .env
```

Заполнить файл:

```env
DATABASE_URL=postgresql+asyncpg://water:secret@db:5432/waterdb
VPS_WS_URL=wss://ваш-домен.com/ws/raspberry
VPS_API_KEY=придумайте-длинный-случайный-ключ
STATION_ID=station_001
POSTGRES_DB=waterdb
POSTGRES_USER=water
POSTGRES_PASSWORD=secret
```

> Если VPS ещё не настроен — оставьте `VPS_WS_URL` пустым. Raspberry будет работать автономно.

### 5.2 Запустить контейнеры

```bash
docker compose up -d
```

Дождаться запуска (~30 секунд). Проверить статус:

```bash
docker compose ps
docker compose logs api --tail=20
```

Ожидаемый вывод: `INFO: Application startup complete.`

### 5.3 Проверить API

```bash
curl http://localhost:8000/health
# → {"status":"ok"}

curl http://localhost:8000/api/stations
# → []
```

## Шаг 6 — Создать станцию

Заменить значения калибровки на реальные размеры вашего бака:

```bash
curl -X POST http://localhost:8000/api/stations \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Основной бак",
    "description": "Садовый резервуар",
    "calibration": {
      "distance_empty": 50.0,
      "distance_full": 5.0,
      "length_cm": 100.0,
      "width_cm": 60.0,
      "height_cm": 45.0
    }
  }'
```

Сохранить `id` из ответа — он понадобится для обновления калибровки через API.

## Шаг 7 — Откалибровать датчик уровня (HC-SR04)

> Эту процедуру удобнее выполнять через экран калибровки в Electron UI (шаг 8).

1. **Полностью опустошить бак**. Подключить Arduino Nano #1 и открыть Serial Monitor — зафиксировать значение `distance_cm`. Это `distance_empty`.

2. **Наполнить бак до максимума**. Зафиксировать `distance_cm`. Это `distance_full`.

3. Измерить внутренние размеры бака (длина, ширина, высота в сантиметрах).

4. Обновить калибровку через API:

```bash
curl -X PATCH http://localhost:8000/api/stations/{ID_СТАНЦИИ} \
  -H "Content-Type: application/json" \
  -d '{
    "calibration": {
      "distance_empty": 50.0,
      "distance_full": 5.0,
      "length_cm": 100.0,
      "width_cm": 60.0,
      "height_cm": 45.0
    }
  }'
```

## Шаг 8 — Запустить Electron GUI (локальный интерфейс)

На Raspberry Pi:

```bash
cd raspberry/electron-ui
npm install
npm run dev
```

При первом запуске `npm install` скачает зависимости (~2–3 минуты). При последующих запусках используйте `npm start`.

Для сборки постоянно работающего приложения:

```bash
npm run pack
# Готовый .deb или исполняемый файл появится в папке dist/
```

## Шаг 9 — Подключить Arduino и проверить работу

1. Подключить Arduino Nano #1 к питанию (убедиться, что ESP8266 питается от внешнего регулятора 3.3В AMS1117, а не от пина 3.3V Nano); ESP-32S подключить к питанию через USB или 5В VIN
2. Убедиться, что все устройства находятся в той же WiFi-сети, что и Raspberry Pi
3. В Electron UI (или через API) должны появиться живые данные:

```bash
# Проверить, что данные приходят
curl http://localhost:8000/api/stations
# В ответе должны обновляться level_pct, volume_l, moisture_pct
```

4. Проверить насосы: нажать «Пуск» в Electron UI — должен щёлкнуть реле и включиться насосы.

## Шаг 10 — Деплой VPS (удалённый доступ)

### 10.1 Подготовить сервер

На VPS выполнить:

```bash
# Установить Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# Перелогиниться

# Склонировать репозиторий на VPS
git clone <url-репозитория>
cd IOT_water_final/vps
```

### 10.2 Настроить домен

В DNS-панели вашего регистратора создать A-запись:
```
@ → IP-адрес VPS
```

Дождаться распространения DNS (от 5 минут до нескольких часов).

### 10.3 Подготовить конфигурацию Traefik

```bash
# Указать свой email для Let's Encrypt
nano traefik/traefik.yml
# Заменить: email: admin@yourdomain.com

# Указать домен в docker-compose.yml
nano docker-compose.yml
# Заменить: yourdomain.com → ваш домен (встречается 2 раза)

# Выставить права на файл сертификатов (обязательно!)
chmod 600 traefik/acme.json
```

### 10.4 Создать `.env` для VPS

```bash
cp .env.example .env
nano .env
```

Сгенерировать хэш пароля (выполнить локально, где установлен Python):

```bash
python3 -c "from passlib.hash import bcrypt; print(bcrypt.hash('ваш_пароль'))"
```

Заполнить `.env`:

```env
DOMAIN=ваш-домен.com
API_KEY=тот-же-ключ-что-в-raspberry-VPS_API_KEY
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=$2b$12$... (вставить сгенерированный хэш)
JWT_SECRET=другой-длинный-случайный-секрет
RASPBERRY_API_URL=http://192.168.1.100:8000
```

> Значение `API_KEY` должно точно совпадать со значением `VPS_API_KEY` в `.env` на Raspberry Pi.

### 10.5 Запустить VPS

```bash
docker compose up -d
```

Проверить, что Traefik получил сертификат (может занять 1–2 минуты):

```bash
docker compose logs traefik --tail=30
# Ожидаемое: "Certificate obtained successfully"

curl https://ваш-домен.com/health
# → {"status":"ok","stations_online":0}
```

### 10.6 Обновить конфигурацию Raspberry

Заполнить `VPS_WS_URL` в `raspberry/.env`:

```env
VPS_WS_URL=wss://ваш-домен.com/ws/raspberry
VPS_API_KEY=тот-же-ключ-что-в-vps-API_KEY
```

Перезапустить контейнер API на Raspberry:

```bash
cd raspberry
docker compose restart api
```

Через несколько секунд в логах VPS должно появиться:
```
INFO: Raspberry connected (station station_001)
```

## Типичные проблемы

### Arduino Nano #1 не подключается к WiFi
- Проверить правильность SSID и пароля в `#define`
- Убедиться, что Raspberry Pi и Arduino в одной сети
- ESP8266 не работает от пина 3.3V Arduino — нужен внешний регулятор AMS1117-3.3

### ESP-32S не подключается к WiFi
- Проверить SSID и пароль в `#define`
- Открыть Serial Monitor (115200 бод) — ошибка будет в логе `[ERROR]`
- Если зависает на "Подключение к WiFi" > 15 сек — ESP.restart() вызывается автоматически
- Убедиться, что сеть 2.4 ГГц (ESP32 не поддерживает 5 ГГц)

### API не стартует (ошибка подключения к БД)
```bash
docker compose logs db
# Если БД ещё поднимается — подождать 30 секунд и перезапустить
docker compose restart api
```

### Traefik не получает сертификат
- Убедиться, что домен указывает на IP VPS (`ping ваш-домен.com`)
- Порт 80 и 443 должны быть открыты в файрволле VPS:
  ```bash
  sudo ufw allow 80 && sudo ufw allow 443
  ```
- Файл `acme.json` должен иметь права 600:
  ```bash
  chmod 600 vps/traefik/acme.json
  ```

### Данные не появляются в VPS
- Проверить, что `VPS_API_KEY` на Raspberry совпадает с `API_KEY` на VPS
- Проверить логи API на Raspberry: `docker compose logs api`
- Проверить логи VPS: `docker compose logs vps-api`

### Уровень показывает 0% или 100% при любом расстоянии
- Выполнить калибровку: убедиться, что `distance_empty > distance_full`
- HC-SR04 направлен строго вниз, без препятствий для ультразвука

## Команды для ежедневного использования

```bash
# Raspberry — проверить статус
cd raspberry && docker compose ps

# Raspberry — посмотреть логи
docker compose logs api -f

# Raspberry — перезапустить
docker compose restart

# VPS — проверить статус
cd vps && docker compose ps

# VPS — посмотреть логи
docker compose logs vps-api -f

# Остановить всё
docker compose down

# Полная очистка (удалит данные БД!)
docker compose down -v
```
