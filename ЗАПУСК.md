# Руководство по запуску проекта

Пошаговая инструкция для полного запуска системы с нуля — от прошивки Arduino до деплоя VPS.

**Топология сети:**
- Все устройства в одной WiFi-сети роутера
- Raspberry Pi имеет статический IP `192.168.1.100`
- Arduino Nano #1 и ESP-32S подключаются к тому же WiFi и стучатся на `192.168.1.100:8000`

---

## Предварительные требования

| Компонент | Что нужно |
|---|---|
| Raspberry Pi 4 | Docker, Docker Compose, Node.js 20+ |
| Компьютер для прошивки (Nano #1) | Arduino IDE 2.x, пакет плат `esp8266`, библиотеки `NewPing` + `ArduinoJson` |
| Компьютер для прошивки (ESP-32S) | Arduino IDE 2.x, пакет плат `ESP32 by Espressif Systems`, библиотеки `arduinoWebSockets` + `ArduinoJson` |
| VPS | Docker, Docker Compose, зарегистрированный домен |
| WiFi-роутер | Все устройства в одной сети 2.4 ГГц |

## Шаг 1 — Записать образ и первый запуск

### 1.1 Записать образ на SD-карту

На компьютере скачать **Raspberry Pi Imager**, выбрать:
- **Ubuntu Server 22.04 LTS (64-bit)** или **Raspberry Pi OS Lite (64-bit)**

Перед записью нажать шестерёнку (⚙) и настроить:
- Включить SSH
- Задать пользователя и пароль
- Указать название и пароль своей WiFi-сети

### 1.2 Подключиться по SSH

После первой загрузки узнать IP Raspberry в панели роутера или командой:

```bash
ping raspberry.local
```

Подключиться:

```bash
ssh project@raspberry.local
```

Обновить систему:

```bash
sudo apt update && sudo apt upgrade -y
```

## Шаг 2 — Назначить статический IP Raspberry Pi

Узнать имя активного WiFi-соединения:

```bash
nmcli con show
# Найти строку с TYPE=wifi и DEVICE=wlan0 — запомнить NAME
```

Назначить статический IP (подставить своё имя соединения):

```bash
sudo nmcli con mod "Название соединения" \
  ipv4.addresses 192.168.1.100/24 \
  ipv4.gateway 192.168.1.1 \
  ipv4.dns "8.8.8.8" \
  ipv4.method manual

sudo nmcli con up "Название соединения"
```

Проверить:

```bash
hostname -I
# Должен показать 192.168.1.100
```

> Если шлюз роутера другой (например 192.168.0.1) — подставь своё значение в `ipv4.gateway`.

## Шаг 5 — Прошить Arduino Nano #1 (насосы + уровень)

### 5.1 Установить библиотеки в Arduino IDE

Открыть **Tools → Manage Libraries**, установить:
- `NewPing` by Tim Eckel
- `ArduinoJson` by Benoit Blanchon (версия 6.x)

### 5.2 Настроить параметры подключения

Открыть `arduino/nano1_pumps/nano1_pumps.ino`, задать свои данные:

```cpp
#define WIFI_SSID      "НазваниеВашейСети"   // ваша WiFi-сеть
#define WIFI_PASS      "ПарольВашейСети"
#define SERVER_HOST    "192.168.1.100"         // статический IP Raspberry Pi
#define SERVER_PORT    8000
#define STATION_ID     "station_001"
```

### 5.3 Загрузить прошивку

1. Подключить Arduino Nano к компьютеру по USB
2. **Tools → Board → Arduino AVR Boards → Arduino Nano**
3. **Tools → Processor → ATmega328P (Old Bootloader)** (если не распознаётся)
4. **Tools → Port** — выбрать нужный порт
5. **Upload** (Ctrl+U)
6. Открыть **Serial Monitor** (9600 бод) — появятся сообщения о подключении к WiFi и WebSocket

## Шаг 6 — Прошить ESP-32S (датчик влажности)

### 6.1 Добавить пакет плат ESP32 в Arduino IDE

**File → Preferences** → в поле «Additional boards manager URLs» добавить:

```
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
```

**Tools → Board → Boards Manager** → найти `ESP32 by Espressif Systems` → установить.

### 6.2 Установить библиотеки

**Tools → Manage Libraries**, установить:
- `arduinoWebSockets` by Markus Sattler
- `ArduinoJson` by Benoit Blanchon (версия 6.x)

### 6.3 Настроить параметры подключения

Открыть `arduino/esp32_moisture/esp32_moisture.ino`, задать свои данные:

```cpp
#define WIFI_SSID      "НазваниеВашейСети"   // та же сеть что и выше
#define WIFI_PASS      "ПарольВашейСети"
#define SERVER_HOST    "192.168.1.100"         // статический IP Raspberry Pi
#define SERVER_PORT    8000
#define STATION_ID     "station_001"
```

### 6.4 Загрузить прошивку

1. Подключить ESP-32S к компьютеру по USB
2. **Tools → Board → ESP32 Arduino → ESP32 Dev Module**
3. **Tools → Upload Speed → 115200**
4. **Tools → Port** — выбрать нужный порт
5. **Upload** (Ctrl+U)
6. Если зависает на `Connecting...` — зажать кнопку **BOOT** на плате, отпустить после начала записи
7. Открыть **Serial Monitor** (115200 бод) — появятся сообщения о подключении

### 6.5 Откалибровать датчик влажности

ESP-32S использует 12-бит АЦП (диапазон 0–4095):

1. Открыть **Serial Monitor** (115200 бод)
2. Дождаться строки `[STATS]` (раз в 60 сек) — записать `Last raw moisture` при сухом датчике → `DRY_VALUE`
3. Поднести датчик к мокрой трубке → записать значение → `WET_VALUE`
4. Вписать в `.ino` и перепрошить:

```cpp
#define DRY_VALUE  800   // ваше значение
#define WET_VALUE  600   // ваше значение
```

## Шаг 7 — Запустить Raspberry Pi (API + БД)

### 7.1 Создать `.env`

```bash
nano .env
```

Заполнить:

```env
DATABASE_URL=postgresql+asyncpg://water:secret@db:5432/waterdb
VPS_WS_URL=
VPS_API_KEY=
STATION_ID=station_001
POSTGRES_DB=waterdb
POSTGRES_USER=water
POSTGRES_PASSWORD=secret
```

> `VPS_WS_URL` оставить пустым пока VPS не настроен — Raspberry работает автономно.

### 7.2 Запустить контейнеры

```bash
docker compose up -d
```

Проверить статус (~30 секунд на запуск):

```bash
docker compose ps
docker compose logs api --tail=20
# Ожидаемый вывод: INFO: Application startup complete.
```

### 7.3 Проверить API

```bash
curl http://192.168.1.100:8000/health
# → {"status":"ok"}

curl http://192.168.1.100:8000/api/stations
# → []
```

## Шаг 8 — Создать станцию

```bash
curl -X POST http://192.168.1.100:8000/api/stations \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Основной бак",
    "description": "Садовый резервуар",
    "calibration": {
      "distance_empty": 50.0,
      "distance_full": 5.0,
      "length_cm": 100.0,
      "width_cm": 60.0,
      "height_cm": 45.0
    }
  }'
```

Сохранить `id` из ответа.

## Шаг 9 — Откалибровать датчик уровня (HC-SR04)

1. Полностью опустошить бак → открыть Serial Monitor → записать `distance_cm` → это `distance_empty`
2. Наполнить бак до максимума → записать `distance_cm` → это `distance_full`
3. Измерить внутренние размеры бака
4. Обновить калибровку:

```bash
curl -X PATCH http://192.168.1.100:8000/api/stations/{ID_СТАНЦИИ} \
  -H "Content-Type: application/json" \
  -d '{
    "calibration": {
      "distance_empty": 50.0,
      "distance_full": 5.0,
      "length_cm": 100.0,
      "width_cm": 60.0,
      "height_cm": 45.0
    }
  }'
```

## Шаг 12 — Деплой VPS (удалённый доступ)

### 12.1 Подготовить сервер

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
git clone <url-репозитория>
cd IOT_water_final/vps
```

### 12.2 Настроить домен

В DNS-панели создать A-запись: `@ → IP-адрес VPS`. Дождаться распространения.

### 12.3 Подготовить Traefik

```bash
nano traefik/traefik.yml      # вставить свой email
nano docker-compose.yml       # вставить свой домен (2 раза)
chmod 600 traefik/acme.json   # обязательно!
```

### 12.4 Создать `.env`

```bash
nano .env
```

Сгенерировать хэш пароля:

```bash
python3 -c "from passlib.hash import bcrypt; print(bcrypt.hash('ваш_пароль'))"
```

Заполнить:

```env
DOMAIN=ваш-домен.com
API_KEY=длинный-случайный-ключ
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=$2b$12$...
JWT_SECRET=другой-длинный-ключ
RASPBERRY_API_URL=http://192.168.1.100:8000
```

### 12.5 Запустить VPS

```bash
docker compose up -d
```

### 12.6 Обновить Raspberry

В `raspberry/.env` заполнить:

```env
VPS_WS_URL=wss://ваш-домен.com/ws/raspberry
VPS_API_KEY=тот-же-ключ-что-в-vps-API_KEY
```

```bash
docker compose restart api
```

## Типичные проблемы

### Arduino/ESP-32S не подключаются к WiFi
- Проверить SSID и пароль в `#define` — должны совпадать с домашней сетью
- Убедиться что сеть **2.4 ГГц** (ESP32 не поддерживает 5 ГГц)
- ESP8266 (Nano #1) не работает от пина 3.3V Arduino — нужен внешний регулятор AMS1117-3.3

### ESP-32S не заливается
- Зажать **BOOT** при `Connecting...`, отпустить после начала записи
- Убедиться: **Tools → Board → ESP32 Arduino → ESP32 Dev Module**
- Если `dfu-util: No DFU capable USB device` — выбрана не та плата
- Попробовать другой USB-кабель

### Raspberry не получает статический IP
```bash
nmcli con show          # найти имя соединения
sudo nmcli con up "Имя" # применить настройки
hostname -I             # проверить IP
```

### API не стартует
```bash
docker compose logs db    # подождать БД
docker compose restart api
```

### Traefik не получает сертификат
- Домен должен указывать на IP VPS: `ping ваш-домен.com`
- Открыть порты: `sudo ufw allow 80 && sudo ufw allow 443`
- Права на файл: `chmod 600 vps/traefik/acme.json`

### Данные не приходят на VPS
- `VPS_API_KEY` на Raspberry должен совпадать с `API_KEY` на VPS
- `docker compose logs api` на Raspberry
- `docker compose logs vps-api` на VPS

### Уровень показывает 0% или 100%
- `distance_empty` должен быть больше `distance_full`
- HC-SR04 направлен строго вниз без препятствий

## Команды для ежедневного использования

```bash
# Raspberry — статус
cd raspberry && docker compose ps

# Raspberry — логи
docker compose logs api -f

# Raspberry — перезапустить
docker compose restart

# VPS — статус
cd vps && docker compose ps

# VPS — логи
docker compose logs vps-api -f

# Остановить всё
docker compose down

# Полная очистка (удалит данные БД!)
docker compose down -v
```
