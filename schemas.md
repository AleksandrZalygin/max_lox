# Схемы подключения железа

## Обзор

Система использует две платы Arduino Nano v3, каждая в паре с WiFi-модулем ESP8266. Всё взаимодействие между железом и Raspberry Pi 4 осуществляется через WebSocket по WiFi.

---

## Arduino Nano #1 — Насосы + датчик уровня

### Компоненты
- Arduino Nano v3
- ESP8266MOD (WiFi)
- HC-SR04 (ультразвуковой датчик расстояния)
- SRD-05VDC-SL-C (двухканальный модуль реле)
- 2× водяных насоса (12В постоянного тока)

### Подключение пинов

| Пин Arduino Nano | Подключить к         | Примечание                              |
|-----------------|----------------------|-----------------------------------------|
| D2 (RX)         | ESP8266 TX           | SoftwareSerial                          |
| D3 (TX)         | ESP8266 RX           | SoftwareSerial (3.3В через делитель)    |
| D5              | Реле IN1             | Активный LOW — LOW = насос ВКЛ          |
| D6              | Реле IN2             | Активный LOW — LOW = насос ВКЛ          |
| D9              | HC-SR04 TRIG         |                                         |
| D10             | HC-SR04 ECHO         |                                         |
| 5V              | HC-SR04 VCC          |                                         |
| 5V              | Реле VCC             |                                         |
| GND             | HC-SR04 GND          |                                         |
| GND             | Реле GND             |                                         |
| GND             | ESP8266 GND          |                                         |

### Питание ESP8266
```
ВАЖНО: НЕ питайте ESP8266 от пина 3.3V Arduino — он не обеспечивает
достаточный ток (ESP8266 потребляет до 250 мА при передаче по WiFi).

Используйте отдельный стабилизатор 3.3В (например, AMS1117-3.3),
запитанный от шины 5В:
  Шина 5В → AMS1117-3.3 IN
  AMS1117-3.3 OUT → ESP8266 VCC (3.3В)
  AMS1117-3.3 GND → общий GND
```

### Делитель напряжения Arduino TX → ESP8266 RX
```
Arduino D3 (логика 5В) → резистор 1 кОм → ESP8266 RX
                                        ↓
                                  резистор 2 кОм
                                        ↓
                                       GND

Напряжение на выходе: 5В × 2/(1+2) = 3.33В (допустимо для ESP8266)
```

### Подключение реле (SRD-05VDC-SL-C)
```
Модуль реле:
  VCC → 5В
  GND → GND
  IN1 → Arduino D5 (активный LOW: LOW = реле сработало = насос ВКЛ)
  IN2 → Arduino D6 (аналогично IN1, оба канала всегда переключаются вместе)

Контакты канала реле (для каждого насоса):
  COM → плюс источника питания 12В
  NO  → провод (+) насоса
  (провод (−) насоса → минус источника питания 12В)

ВАЖНО: Реле активное LOW. Когда IN1/IN2 = HIGH → реле отключено (насос ВЫКЛ).
       Когда IN1/IN2 = LOW → реле сработало (насос ВКЛ).
       Оба насоса ВСЕГДА переключаются одновременно — никогда по отдельности.
```

### Схема питания
```
Блок питания 12В постоянного тока:
  12В → клеммы COM реле (к насосам)
  12В → вход понижающего преобразователя (step-down до 5В)

Понижающий преобразователь (например, модуль LM2596):
  IN:  12В
  OUT: 5В → Arduino Nano VIN, Реле VCC, HC-SR04 VCC

AMS1117-3.3 (от 5В):
  IN:  5В
  OUT: 3.3В → ESP8266 VCC
```

### Монтаж HC-SR04
- Закрепить датчик на крышке бака, направив вниз к поверхности воды
- Расположить по центру, чтобы избежать отражений от стенок бака
- Датчик измеряет расстояние от крышки до поверхности воды
- Пустой бак: большее значение расстояния (например, 50 см)
- Полный бак: меньшее значение расстояния (например, 5 см)

## ESP-32S — Датчик потока слива

### Компоненты
- ESP-32S (встроенный WiFi + 12-бит АЦП, не требует отдельного модуля ESP8266)
- Ёмкостный датчик влажности почвы (используется как детектор потока воды в трубке)

### Подключение пинов

| Пин ESP-32S | Подключить к                  | Примечание                              |
|-------------|-------------------------------|-----------------------------------------|
| GPIO34      | Датчик влажности AOUT         | Input-only пин, идеален для АЦП        |
| 3.3V        | Датчик влажности VCC          | Встроенный LDO ESP-32S достаточен (≤20мА)|
| GND         | Датчик влажности GND          |                                         |
| USB / 5V    | Питание ESP-32S               | Через USB или пин VIN/5V платы          |

> **Преимущество перед Nano #2 + ESP8266:** ESP-32S имеет встроенный WiFi и 12-бит
> АЦП — не нужен отдельный ESP8266, SoftwareSerial, делитель напряжения и
> ручное формирование WebSocket-фреймов.

### Значения калибровки (12-бит АЦП, диапазон 0–4095)
```
DRY_VALUE  = 2800   (датчик в воздухе / сухая труба — воды нет)
WET_VALUE  = 1400   (датчик полностью погружён / трубка с водой)

moisture_pct = constrain(map(raw, DRY_VALUE, WET_VALUE, 0, 100), 0, 100)

Процедура калибровки:
  1. Запустите Serial Monitor (115200 бод)
  2. Поднесите датчик к сухой трубке — запишите raw в логе [STATS]
  3. Поднесите к мокрой трубке (или погрузите в воду) — запишите raw
  4. Подставьте значения в #define DRY_VALUE / WET_VALUE
  5. Перепрошейте и убедитесь что moisture_pct = 0% в воздухе, 100% в воде
```

### Расположение датчика
- Вставить датчик в трубку слива после ручного крана
- Датчик определяет наличие и примерную интенсивность потока воды
- Высокий moisture_pct (>30%) → вода течёт по трубке → нормальный слив
- Низкий moisture_pct (<30%) → воды в трубке нет → если уровень падает — протечка

### Питание
- ESP-32S питается от USB (для разработки) или от 5В через пин VIN/5V
- Встроенного LDO 3.3В достаточно для ёмкостного датчика влажности
- Внешний стабилизатор AMS1117 не нужен (в отличие от схемы с ESP8266)
- Делитель напряжения не нужен (нет SoftwareSerial)

## Топология сети

```
┌─────────────────────────────────────────┐
│          WiFi-роутер (локальная сеть)   │
│              192.168.1.0/24             │
└───────┬──────────────┬──────────────────┘
        │              │              │
        ▼              ▼              ▼
  ESP8266 #1      ESP-32S       Raspberry Pi 4
  (Nano1)         (влажность)   Статический IP: 192.168.1.100
  DHCP            DHCP          Порт 8000: FastAPI
                                Порт 3000: Electron (опционально)

WebSocket-соединения:
  ESP8266 #1 (Nano1)  → ws://192.168.1.100:8000/ws/arduino  (device: "nano1")
  ESP-32S             → ws://192.168.1.100:8000/ws/arduino  (device: "esp32")

Raspberry Pi → VPS:
  wss://ваш-домен.com/ws/raspberry
```

### Настройка статического IP на Raspberry Pi
```bash
# /etc/dhcpcd.conf
interface eth0
static ip_address=192.168.1.100/24
static routers=192.168.1.1
static domain_name_servers=8.8.8.8
```
